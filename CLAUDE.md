# Reservation System SASS - Claude Development Guide

## Project Overview
Building an AI-powered SaaS reservation system with Next.js, TypeScript, Supabase, and Tailwind CSS.

## Top 5 Integrated Skills

### 1. Superpowers 🚀
**Purpose**: Maximize development velocity and workflow efficiency
- Fast feature implementation
- Smart code organization
- Quick debugging and fixes

### 2. Obsidian 📚
**Purpose**: Document architecture, requirements, and system design
- Keep your reservation logic documented
- Link requirements to code
- Build a knowledge base as you develop

### 3. UI UX Pro Max 🎨
**Purpose**: Design exceptional user experiences for booking flows
- Design reservation interface
- User journey optimization
- Accessibility & usability

### 4. Impeccable ✨
**Purpose**: Maintain high code quality standards
- Code reviews and refinement
- Pattern consistency
- Performance optimization

### 5. Caveman 📊
**Purpose**: Analyze reservation data and system performance
- Track booking metrics
- Identify patterns in user behavior
- Performance analytics

## Quick Start Commands

```bash
# Feature development (Superpowers)
/superpowers [feature-description]

# Document system design (Obsidian)
/obsidian [topic]

# Design UX flows (UI UX Pro Max)
/ui-ux-pro-max [screen/flow]

# Code review & refinement (Impeccable)
/impeccable [review-focus]

# Data analysis (Caveman)
/caveman [analysis-type]
```

## Project Structure
- `/src/app` - Next.js pages & routes
- `/src/components` - React components
- `/src/lib` - Utilities & Supabase integration
- `/supabase` - Database schema & migrations

## Development Workflow
1. **Plan** with Obsidian (document requirements)
2. **Build** with Superpowers (fast implementation)
3. **Design** with UI UX Pro Max (user experience)
4. **Review** with Impeccable (code quality)
5. **Analyze** with Caveman (metrics & insights)
